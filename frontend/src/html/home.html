Capabilities

A full behavioral security layer for public networks

Six detection engines working together, trained on real-world access-point behavior.

AI Trust Score

A single 0–100 score generated by our ML pipeline, combining signal, protocol and behavioral features into one instant verdict.

Evil Twin Detection

Flags rogue access points impersonating legitimate SSIDs by comparing BSSID history, signal drift and beacon fingerprints.

DNS Spoofing Detection

Continuously validates DNS responses against trusted resolvers to catch man-in-the-middle redirection attempts.

Real-time Monitoring

Live packet and connection telemetry streamed to your dashboard, so threats surface the moment they appear.

Behavioral Analysis

Learns normal access-point behavior over time and detects anomalies in traffic patterns, latency and reconnection cycles.

Privacy Preserving

All feature extraction happens on-device where possible; raw traffic content is never stored or transmitted.

About the project

How the detection pipeline works

From raw network signals to a single trust decision, in four stages.

1

Signal & Metadata Capture

SSID, BSSID, RSSI, encryption type, channel and beacon intervals are collected passively during a scan.

2

Feature Engineering

Raw signals are transformed into behavioral features — signal variance, response latency, DNS consistency and more.

3

ML Classification

A trained ensemble model (Random Forest / XGBoost) scores the network and flags evil-twin or spoofing patterns.

4

Trust Score & Recommendation

Results are compiled into a 0–100 trust score with a clear, plain-language recommendation before you connect.
